시스템 개요

A2Z 자율주행차는 다양한 센서와 ECU를 TSN(Time-Sensitive Networking) 기반으로 연결하여 실시간성과 신뢰성을 보장합니다.

핵심 플랫폼

  • ACU_NO: NVIDIA Jetson Orin 기반
  • ACU_IT: Intel Tiger Lake 기반
  • TSN Switch: Microchip Automotive Ethernet

주요 기술

  • IEEE 802.1Qbv (Time Aware Shaper)
  • IEEE 802.1Qav (Credit Based Shaper)
  • IEEE 802.1CB (Frame Replication and Elimination)
  • IEEE 802.1AS (Generalized PTP)

네트워크 아키텍처

옵션 A: 단일 스위치 구성

센서 → TSN 스위치 → ACU_IT

장점
  • 최소 홉(1-2)
  • 최소 지연
  • 구현 간단
단점
  • 단일 장애점
  • FRER 불가
  • 확장성 제한

옵션 B: 듀얼 엣지 + 중앙 집선

좌/우 엣지 스위치 + 중앙 집선 → ACU_IT

장점
  • 포트 수 유연
  • 케이블 최적화
  • 부분 이중화
단점
  • 홉 증가(2-3)
  • 중앙 집선 SPOF

옵션 C: 3스위치 메시 + ACU 듀얼홈

완전 메시 토폴로지, ACU 이중경로

장점
  • 진정한 경로 이중화
  • FRER 완전 지원
  • 고가용성
단점
  • 구성 복잡
  • 비용 증가
  • 관리 오버헤드

고장 시나리오 코드 체계

[SCN]-[LOC]-[DEV]-[FLT] (+[RES])

코드 구성요소

구분 코드 의미
SCN A-H 손실, 지연, 순서, 손상, 동기, 장비고장, 전원, 구성
LOC ZF/ZB/ZL/ZR Front/Rear/Left/Right 존
DEV LP40/LG32/MR35/CAMV 센서 및 장비 식별
FLT LNK/QDR/CON/CRC... 세부 원인

해결 방안

FRR FRER 무중단 복구
RER 즉시 경로 전환
CBS+ CBS 재프로파일
PSFP 스트림 폴리싱
PTP 동기 복구
RST 재시작/재부팅
DGR 성능저하모드
ISO 격리/바이패스
RBK 설정 롤백
SSC 안전정지
